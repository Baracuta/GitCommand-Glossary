Git Command Glossary

A glossary for 10 of the most common Git Commands

Git Add
Moves changes from the working directory to the staging area. Prepare a snapshot before committing it to the official history.
Git Branch
General-purpose branch administration tool. Create isolated developments environments within a repository.
Git Checkout
Used to check out old commits and file revisions, as well as navigate existing branches.
Git Clean
Removes untracked files from the working directly. Logical counterpart to git reset, which (generally) only works on tracked files.
Git Clone
Creates a copy of an existing git repository. The most common way for developers to obtain a working copy of a central repository.
Git Commit
Takes the staged snapshot and commits it to the project history. Combined with git add, this defines the basic workflow for all git users.
Git Config
A convenient way to set configuration options for your git installation. You'll probably only need to use this immediately after installing git on a new machine.
Git Init
Initializes a new git repository. If you wish to place a project under revision control, this is the first command you need to learn.
Git Pull
Downloads a branch from a remote repository,then immediately merges it into the current branch. Git counterpart to svn update.
Git Push
Moves a local branch to another repository, which serves as a convenient way to publish contributions. Like svn commit, but sends a series of commits instead.
